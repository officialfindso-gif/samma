# Руководство по разработке

Этот документ содержит информацию для разработчиков, участвующих в разработке проекта.

## Структура проекта

### Backend (Django)

#### Основные компоненты
- `core/` - основные настройки Django
  - `settings.py` - настройки проекта
  - `urls.py` - глобальные URL-маршруты
  - `wsgi.py` - WSGI-конфигурация
- `saas/` - основное приложение
  - `models.py` - все модели данных
  - `views.py` - API-представления
  - `serializers.py` - сериализаторы DRF
  - `admin.py` - настройки админ-панели
  - `tasks.py` - асинхронные задачи Celery
  - `llm.py` - интеграция с LLM

#### Модели данных
- `Workspace` - воркспейсы, основная единица организации
- `WorkspaceMember` - участники воркспейсов
- `Post` - посты для обработки
- `Prompt` - шаблоны промптов
- `SocialAccount` - учетные записи соцсетей
- `SubscriptionPlan` - тарифные планы
- `UserSubscription` - подписки пользователей
- `SubscriptionFeature` - особенности тарифных планов

#### API-эндпоинты
Все API-эндпоинты зарегистрированы в `core/urls.py` через DRF Routers.

#### Асинхронные задачи
Используется Celery для обработки длительных операций, таких как генерация контента через LLM.

### Frontend (Next.js)

#### Основные компоненты
- `src/app/` - страницы приложения (App Router)
- `src/lib/api.ts` - клиентская обертка для API
- `src/components/` - переиспользуемые компоненты (если есть)

#### Страницы
- `/login` - страница аутентификации
- `/app` - основное приложение
- `/prompts` - управление промптами
- `/profile` - профиль пользователя

#### Управление состоянием
Состояние хранится в компонентах React и в localStorage (токены аутентификации).

## Соглашения по коду

### Backend
- Используем Django Code Standards
- Для импортов используем следующий порядок:
  1. Стандартная библиотека Python
  2. Сторонние библиотеки
  3. Импорты из проекта
- Для именования классов используем PascalCase
- Для функций и переменных используем snake_case

### Frontend
- Используем TypeScript строго
- Для компонентов используем PascalCase
- Для функций и переменных используем camelCase
- Все строки интернационализируем (в будущем)

## Работа с базой данных

### Миграции
Для создания миграций используйте:
```bash
python manage.py makemigrations
```

Для применения миграций:
```bash
python manage.py migrate
```

### Создание суперпользователя
```bash
python manage.py createsuperuser
```

## Работа с асинхронными задачами

### Запуск Celery worker
```bash
celery -A core worker -l info
```

### Запуск Celery beat (для периодических задач)
```bash
celery -A core beat -l info
```

## Безопасность

### Проверка доступа
- Все API-эндпоинты защищены JWT-аутентификацией
- Проверка доступа к объектам происходит на уровне воркспейсов
- Используются DRF permissions

### Валидация данных
- Валидация входных данных через сериализаторы DRF
- Проверка бизнес-логики в методах `validate_*` сериализаторов

## Деплой

### Подготовка к деплою
1. Обновите зависимости в `requirements.txt`
2. Проверьте настройки в `core/settings.py` для продакшена
3. Убедитесь, что все миграции применены
4. Запустите тесты

### CI/CD (планируется)
- Автоматические тесты
- Проверка качества кода
- Деплой на staging/production

## Расширение функциональности

### Добавление новых моделей
1. Определите модель в `saas/models.py`
2. Создайте сериализатор в `saas/serializers.py`
3. Создайте ViewSet в `saas/views.py`
4. Зарегистрируйте в `core/urls.py`
5. Добавьте в админ-панель в `saas/admin.py`
6. Создайте и примените миграции

### Добавление новых API-эндпоинтов
1. Создайте ViewSet или View в `saas/views.py`
2. Добавьте сериализатор в `saas/serializers.py`
3. Зарегистрируйте в `core/urls.py`

### Добавление новых страниц во frontend
1. Создайте новую директорию в `src/app/`
2. Добавьте `page.tsx` в эту директорию
3. Реализуйте логику с использованием API-функций из `src/lib/api.ts`

## Часто задаваемые вопросы

### Как добавить нового пользователя?
Через админ-панель Django или через `createsuperuser`.

### Как изменить структуру базы данных?
1. Измените модель в `saas/models.py`
2. Создайте миграцию: `python manage.py makemigrations`
3. Примените миграцию: `python manage.py migrate`

### Как добавить новую интеграцию с LLM?
1. Добавьте необходимую логику в `saas/llm.py`
2. Создайте новую задачу в `saas/tasks.py` если требуется асинхронная обработка
3. Обновите соответствующие ViewSet'ы и сериализаторы
