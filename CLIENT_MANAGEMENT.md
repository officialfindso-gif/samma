# Документация по управлению клиентами

## Обзор

Система управления клиентами позволяет организовать работу с несколькими проектами, отслеживать активности и быстро переключаться между клиентами.

## Основные возможности

### 1. Список клиентов (`/app/clients`)

**Функции:**
- Просмотр всех клиентов в виде карточек
- Создание нового клиента
- Редактирование данных клиента
- Удаление клиента
- Быстрый переход к постам клиента
- Детальный просмотр информации о клиенте

**Отображаемая информация на карточке:**
- Название проекта и клиента
- Контактная информация
- Заметки о клиенте
- Теги для категоризации
- Количество постов
- Количество мест в команде
- Дата создания

### 2. Детальный просмотр клиента (`/app/clients/[id]`)

**Отображаемая информация:**
- **Шапка:** Название, контакт, теги
- **Статистика:** Посты, места в команде, дни сотрудничества, количество активностей
- **Заметки:** Полный текст заметок о клиенте
- **Активности:** Полный список всех взаимодействий с клиентом
- **Информация о проекте:** Даты создания/обновления, цвет, ID workspace

### 3. Управление активностями

**Типы активностей:**
- 📝 Заметка
- 📞 Звонок
- 🤝 Встреча
- 📧 Email
- ✨ Создан пост
- ✅ Утверждён пост
- 💰 Оплата
- 📌 Другое

**Создание активности:**
1. Откройте страницу детального просмотра клиента
2. Нажмите кнопку "➕ Добавить"
3. Выберите тип активности
4. Введите заголовок (обязательно)
5. Введите описание (опционально)
6. Нажмите "➕ Создать"

**Отображение активности:**
- Иконка типа активности
- Заголовок и описание
- Имя автора
- Дата и время создания
- Кнопка удаления (видна при наведении)

### 4. Теги

**Назначение:** Категоризация клиентов для быстрой фильтрации и поиска.

**Формат ввода:** Теги вводятся через запятую в поле "Теги" при создании/редактировании клиента.

**Пример:** `срочно, премиум, активный`

**Отображение:** Теги показываются как цветные бейджи с префиксом `#`.

### 5. Быстрое переключение между клиентами

**Где:** Выпадающее меню в шапке приложения (Header)

**Как использовать:**
1. Нажмите на кнопку с иконкой клиента в шапке
2. Выберите клиента из списка
3. Автоматический переход к постам выбранного клиента

**Особенности:**
- Показывает только клиентские workspace (is_client=true)
- Цветная граница для визуального различения
- Отображается количество постов каждого клиента

## База данных

### Модель WorkspaceActivity

```python
class WorkspaceActivity(models.Model):
    workspace = models.ForeignKey(Workspace, on_delete=models.CASCADE)
    activity_type = models.CharField(max_length=20, choices=[
        ('note', 'Заметка'),
        ('call', 'Звонок'),
        ('meeting', 'Встреча'),
        ('email', 'Email'),
        ('post_created', 'Создан пост'),
        ('post_approved', 'Утверждён пост'),
        ('payment', 'Оплата'),
        ('other', 'Другое'),
    ])
    title = models.CharField(max_length=200)
    description = models.TextField(blank=True)
    created_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True)
    created_at = models.DateTimeField(auto_now_add=True)
```

### Расширенная модель Workspace

**Добавленные поля:**
- `is_client` - Флаг клиентского workspace
- `client_name` - Название компании/ФИО клиента
- `client_contact` - Email/телефон/Telegram
- `client_notes` - Заметки о клиенте
- `color` - Цвет для визуального различения
- `tags` - Теги через запятую
- `updated_at` - Дата последнего изменения

**Методы:**
- `get_tags_list()` - Возвращает список тегов из CSV-строки

## Рабочий процесс

### Типичный сценарий работы с клиентом:

1. **Создание клиента**
   - Перейти на `/app/clients`
   - Нажать "➕ Добавить клиента"
   - Заполнить форму (название, контакт, заметки, теги, цвет)
   - Сохранить

2. **Создание постов**
   - Использовать быстрый переключатель в Header
   - ИЛИ нажать "🎯 Посты" на карточке клиента
   - Создавать посты как обычно (они автоматически привязываются к workspace клиента)

3. **Отслеживание взаимодействий**
   - Открыть детальную страницу клиента (📊 Подробнее)
   - Добавлять активности при каждом взаимодействии:
     - Звонки с клиентом
     - Встречи
     - Отправка материалов по email
     - Оплаты
     - Создание/утверждение постов

4. **Анализ работы**
   - Статистика на dashboard клиента:
     - Сколько постов создано
     - Сколько дней сотрудничества
     - История всех активностей
   - Используйте теги для фильтрации клиентов (например, показать только "срочно" или "vip")

## Пермишены

**Доступ к клиентам:** Пользователь видит только те workspace, в которых он является участником (через WorkspaceMember).

**Создание активностей:** Автоматически сохраняется `created_by = request.user`.

**Редактирование/удаление:** Доступно всем участникам workspace. Для более гранулярного контроля можно добавить проверку роли (owner/admin/editor).
