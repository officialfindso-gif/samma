# 🛠️ Система настройки времени автоматического парсинга

## Что реализовано

### Backend (Django)

1. **Модель `SystemSettings`** - хранит настройки системы:
   - `auto_scraping_enabled` - включен ли автопарсинг
   - `scraping_hour` - час запуска (0-23)
   - `scraping_minute` - минута запуска (0-59)
   - `last_scraping_check` - время последней проверки

2. **API эндпоинты** (`/api/settings/`):
   - `GET /api/settings/current/` - получить текущие настройки
   - `PATCH /api/settings/1/` - обновить настройки

3. **Обновленная Celery задача `scrape_competitor_posts()`**:
   - Запускается каждый час (в 00 минут)
   - Проверяет `SystemSettings` перед выполнением
   - Если `auto_scraping_enabled = False` - пропускает
   - Проверяет текущее время с заданным (допуск ±5 минут)
   - Выполняет парсинг только в настроенное время

4. **Celery Beat schedule**:
   - Задача запускается каждый час: `crontab(minute=0)`
   - Сама задача решает, парсить или нет, на основе настроек

### Frontend (Next.js)

1. **Страница `/app/settings`**:
   - Включение/выключение автопарсинга
   - Настройка времени (часы и минуты)
   - Отображение последней проверки
   - Визуальное отображение текущего времени

2. **Ссылка в Sidebar**: "⚙️ Настройки"

## Как использовать

### 1. Откройте страницу настроек

```
http://localhost:3000/app/settings
```

### 2. Измените время парсинга

- Выберите час (0-23)
- Выберите минуту (0-59)
- Нажмите "💾 Сохранить настройки"

### 3. Перезапустите Celery Beat

**⚠️ ВАЖНО:** После изменения времени необходимо перезапустить Celery Beat:

```bash
# Остановите текущий процесс Celery Beat (Ctrl+C)
# Затем запустите снова:
cd backend
python -m celery -A core beat -l info
```

### 4. Проверьте работу

Celery Beat будет запускать задачу каждый час, а задача сама решит, парсить ли контент на основе настроек.

## Примеры использования

### Пример 1: Парсинг в 9:00

1. Установите время: `09:00`
2. Включите автопарсинг
3. Сохраните
4. Перезапустите Celery Beat

Теперь система будет парсить конкурентов каждый день в 9:00 (±5 минут).

### Пример 2: Парсинг в 14:30

1. Установите время: `14:30`
2. Включите автопарсинг
3. Сохраните
4. Перезапустите Celery Beat

Парсинг будет происходить в 14:30.

### Пример 3: Отключение автопарсинга

1. Снимите галочку "Включить автоматический парсинг"
2. Сохраните

Теперь автоматический парсинг не будет выполняться. Вы всё ещё можете запускать его вручную через кнопки в разделе "Конкуренты".

## Технические детали

### Как это работает?

1. **Celery Beat** запускает задачу `scrape_competitor_posts()` каждый час
2. Задача читает настройки из `SystemSettings.get_settings()`
3. Проверяет:
   - Включен ли автопарсинг (`auto_scraping_enabled`)
   - Совпадает ли текущее время с настроенным (±5 минут)
4. Если условия выполнены - парсит всех активных конкурентов
5. Обновляет `last_scraping_check` после каждой проверки

### Допуск времени

Используется допуск ±5 минут, потому что Celery Beat может запуститься не ровно в 00 минут (зависит от нагрузки системы).

Например, если настроено время `09:00`, задача выполнится если текущее время:
- 08:55 - 09:05

### Singleton настроек

Модель `SystemSettings` реализована как singleton - всегда существует только один объект с `id=1`.
